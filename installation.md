# Installation — DoctoRDV

## Objectif

Ce document présente l’environnement prévu pour développer et tester l’application DoctoRDV.

À cette étape, l’application n’est pas encore développée. Le but est de préparer les prérequis techniques avant de commencer le code.

## Environnement prévu

Le projet sera développé en local avec une stack simple :

- PHP ;
- MySQL ;
- HTML / CSS ;
- Bootstrap ;
- JavaScript simple si nécessaire ;
- Git ;
- GitHub ;
- Visual Studio Code ;
- Laragon ou un environnement local équivalent.

## Choix d’architecture

Pour la première version, le projet sera développé en PHP simple, sans framework complexe.

Ce choix permet de rester réaliste pour un projet BTS SIO SLAM et de bien comprendre :

- les pages PHP ;
- les formulaires ;
- les traitements ;
- les requêtes SQL ;
- la connexion à la base de données ;
- la gestion des rôles.

## Étapes prévues plus tard

Les étapes techniques prévues seront :

1. installer ou vérifier l’environnement local ;
2. placer le projet dans un dossier de développement ;
3. créer une base de données MySQL ;
4. créer les premières pages PHP ;
5. connecter l’application à la base de données ;
6. tester la réservation d’un rendez-vous ;
7. versionner le projet avec Git ;
8. envoyer le projet sur GitHub.
