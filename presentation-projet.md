# Présentation du projet — DoctoRDV

## Contexte

DoctoRDV est une application web développée dans le cadre du BTS SIO option SLAM.

Elle permet de gérer les rendez-vous entre des patients et des professionnels de santé.

Le projet est réalisé comme support principal de l'épreuve E6.

## Objectif

L'objectif est de développer une application simple permettant :

- la gestion des utilisateurs ;
- la gestion des patients ;
- la gestion des médecins ;
- la prise de rendez-vous ;
- la consultation des rendez-vous ;
- la gestion des rôles.

## Utilisateurs

L'application est destinée à :

- l'administrateur ;
- les médecins ;
- les secrétaires.

Les patients n'ont pas d'espace personnel dans cette première version.

## Fonctionnalités prévues

- authentification ;
- gestion des rôles ;
- gestion des patients ;
- gestion des médecins ;
- prise de rendez-vous ;
- modification d'un rendez-vous ;
- suppression d'un rendez-vous ;
- tableau de bord.

## Technologies

Le projet utilise :

- PHP ;
- MySQL ;
- HTML ;
- CSS ;
- Bootstrap ;
- JavaScript ;
- Git ;
- GitHub.

## Limites

Le projet reste volontairement adapté au niveau BTS SIO.

Il ne prévoit pas :

- paiement en ligne ;
- notifications SMS ;
- application mobile ;
- téléconsultation ;
- agenda synchronisé.
