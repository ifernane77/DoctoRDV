# Conventions de nommage — DoctoRDV

## Objectif

Ce document définit les règles de nommage utilisées dans le projet DoctoRDV.

L’objectif est de garder un projet propre, cohérent et facile à comprendre.

## Fichiers Markdown

Les fichiers Markdown sont écrits en minuscules avec des tirets.

Exemples :

- `journal-de-bord.md`
- `cas-utilisation.md`
- `architecture-et-flux.md`
- `conventions-nommage.md`

## Dossiers

Les dossiers techniques sont nommés simplement.

Exemples :

- `database`
- `src`
- `tests`
- `preuves`

## Futures tables SQL

Les futures tables seront nommées en minuscules.

Exemples :

- `users`
- `roles`
- `patients`
- `praticiens`
- `specialites`
- `creneaux`
- `rendez_vous`
