# Objectifs du projet — DoctoRDV

## Objectif général

Créer une application web simple permettant de gérer des rendez-vous médicaux fictifs pour une maison médicale appelée MediCentre Nova.

## Problème identifié

La gestion des rendez-vous par téléphone peut entraîner :

- des erreurs de planning ;
- des oublis ;
- une mauvaise visibilité sur les disponibilités ;
- une perte de temps pour l’administration ;
- une difficulté à suivre les rendez-vous par praticien.

## Objectifs fonctionnels

L’application doit permettre de :

- consulter les praticiens ;
- consulter les disponibilités ;
- réserver un rendez-vous ;
- annuler un rendez-vous ;
- connecter les utilisateurs ;
- gérer les rôles ;
- gérer les praticiens ;
- gérer les créneaux ;
- consulter un planning praticien ;
- administrer les rendez-vous.

## Objectifs techniques

Le projet doit permettre de mettre en pratique :

- PHP ;
- MySQL ;
- HTML / CSS ;
- Bootstrap ;
- JavaScript simple ;
- Git et GitHub ;
- une structure de projet organisée ;
- une documentation Markdown.

## Objectifs pédagogiques BTS SIO

Ce projet doit me permettre de travailler les compétences suivantes :

- analyser un besoin ;
- concevoir une solution applicative ;
- modéliser une base de données ;
- développer des interfaces web ;
- gérer des données ;
- tester une application ;
- documenter une réalisation professionnelle ;
- présenter un projet à l’oral.

## Limites du projet

Pour rester réaliste, la première version ne prévoit pas :

- de paiement en ligne ;
- de données médicales réelles ;
- de SMS automatique ;
- de visioconférence ;
- de synchronisation avec un agenda externe.

Les données utilisées seront fictives.
