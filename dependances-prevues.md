# Dépendances prévues — DoctoRDV

## Objectif

Ce document liste les outils et dépendances qui seront utiles pour développer DoctoRDV plus tard.

À cette étape, je ne les installe pas encore toutes.

Le but est simplement d’identifier ce dont j’aurai besoin.

---

## Outils prévus

| Outil | Utilisation prévue |
|---|---|
| Visual Studio Code | Écriture du code et de la documentation |
| Git | Suivi des versions du projet |
| GitHub | Sauvegarde du projet et preuves BTS |
| Laragon | Environnement local PHP/MySQL |
| Navigateur web | Test des pages |
| MySQL | Base de données |
| PHP | Développement côté serveur |
| Bootstrap | Mise en forme simple de l’interface |

---

## Pourquoi ces outils ?

J’ai choisi une stack simple pour rester cohérent avec un projet BTS SIO SLAM.

DoctoRDV doit rester compréhensible et présentable.

L’objectif n’est pas de faire une application trop complexe, mais une application fonctionnelle avec :

- des praticiens ;
- des disponibilités ;
- des rendez-vous ;
- des utilisateurs ;
- des rôles.

---

## Dépendances à installer plus tard

Les éléments à installer ou vérifier plus tard seront :

1. Laragon ;
2. PHP ;
3. MySQL ;
4. Bootstrap ;
5. éventuellement une bibliothèque PDF ou calendrier si le projet en a besoin.
