# Backlog — DoctoRDV

## Objectif du backlog

Ce backlog permet de suivre les fonctionnalités prévues pour l’application DoctoRDV.

Il sera mis à jour progressivement pendant le projet.

## Fonctionnalités prioritaires

| Priorité | Fonctionnalité | Description | Statut |
|---|---|---|---|
| Haute | Page d’accueil | Présenter l’application et orienter l’utilisateur | À faire |
| Haute | Liste des praticiens | Afficher les praticiens disponibles | À faire |
| Haute | Disponibilités | Afficher les créneaux disponibles | À faire |
| Haute | Réservation | Permettre à un patient fictif de réserver un rendez-vous | À faire |
| Haute | Authentification | Permettre la connexion des utilisateurs | À faire |
| Haute | Gestion des rôles | Prévoir les rôles patient, praticien et administrateur | À faire |
| Moyenne | Planning praticien | Permettre au praticien de consulter ses rendez-vous | À faire |
| Moyenne | Annulation | Permettre l’annulation d’un rendez-vous | À faire |
| Moyenne | Administration | Gérer les praticiens, créneaux et rendez-vous | À faire |

## Version minimale attendue

La version minimale doit permettre de :

- consulter les praticiens ;
- voir les créneaux disponibles ;
- réserver un rendez-vous ;
- consulter les rendez-vous enregistrés ;
- administrer les données principales.

## Version améliorée

La version améliorée pourra permettre de :

- gérer des rôles plus précis ;
- annuler un rendez-vous ;
- filtrer les praticiens par spécialité ;
- améliorer l’interface avec Bootstrap ;
- ajouter des contrôles de formulaire ;
- ajouter des tests fonctionnels.
